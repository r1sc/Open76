Introduction

I've written this guide to explain the methods I have developed to create custom paintjobs for vehicles in the game Interstate76. In the course of the guide I will explain what files are used to define the textures, and how to adapt these files to create custom paintjobs.

This is reasonably advanced hacking and may seem daunting at first, but I have devised a series of step-by-step exercises that should enable you to understand the basic principles. I won’t pretend it will be very easy, and I will guarantee you will probably get very frustrated at various stages, but if you persist you will be able to create your own customs.

To follow this guide I advise printing it out…but before you do that finish reading this introduction as you need some files...

Before we start some things need...

You will need some files that are refered to in this guide. You can open a zip with them in by clicking HERE. From now on the contents of this zip will be referred to as "the "files" folder that accompanies this guide"

To open the files for viewing you will need a hex editor - I advise using Break Point Software's Hex Workshop. This is editor used in the examples in this guide, and a fully functioning evaluation copy is available for download from Break Point's website (http://www.bpsoft.com)

It is possible to create custom textures using only a hex editor by editing an image pixel by pixel (This is how I made the first customs, but this is extremely time consuming and takes great patience).

I now have an easier method that allows you to create your textures in the more regular manner of using a paint program (though some hexing is still necessary).

The paint program used in these examples is JASC's Paint Shop Pro, an evaluation copy of which is also available for download. ( http://www.jasc.com )[NOTE version 4.10 allows continued use after the trial 30 days, whereas the most recent version, 5.0, is limited to 30 days use]

The original texture files (along with many other types of file) are contained within a file called I76.ZFS. This is a huge file (60.8 MB) made up of 3927 different files back to back. The ZFS is located at C:/Program Files/Activision/I76. To extract files from the ZFS you need pack76 a utility made by ReVoLt and bIGfreek that are available to download from Smidge's website. (http://www.smidge.com/files/html/miscfiles.htm)

One of the most important things to know when modifying Interstate76 is that any file (or copy of a file) that is in the Addon folder will be used by the game, even if a file of the same name is in the ZFS. The Addon folder is located at C:/Program Files/Activision/I76/Addon

For the purposes of this guide I will refer to working with the original i76 game - this is because the files in Nitro/Gold Edition are encrypted, and so inaccessible. Files extracted from the original i76 can be placed in the Gold or Nitro Addon and will usually function normally.

In case you do not have a copy of the original I76 there are copies of the files extracted from I76.ZFS that are used in the exercises. They can be found in the "files" folder that accompanies this guide.

 

Finding out what file does what...

First of all we need to open a VCF file. VCF stands for Vehicle Configuration File. These files determine the type, paintjob, wheels, armour, suspension, weapons and specials settings of any given vehicle. This information is conveyed by either a value in hex (as in the value of armour) or by referring to another file (which is how the game knows which paintjob your car has).

When you create a new variant of a vehicle a new VCF file is created in your Addon folder (C:/Program Files/Activision/I76/Addon).

The VCFs that define the default variants available in auto-melee, the trip, and all the a.i. vehicles are contained in the ZFS.

In the following exercise we will use the file VPPIRNA1.VCF as an example. This is the first variant of the Picard Piranha, in familiar orange and black…

 

Exercise 1 (Altering a VCF)

  • Extract the file VPPIRNA1.VCF from I76.ZFS using PACK76
  • Open the file VPPIRNA1.VCF with Hex Workshop

You should see the following...

I've marked in red the reference to the paintjob file - which in this case is PIRANHA1.VTF.

VTF stands for Vehicle Texture File. There is one VTF file for each paintjob in the game, and all these files are within the ZFS and can be extracted with Pack76.

If you configure the Piranha within I76 you only have an option of 3 different paintjobs; that is the ones designed for the Piranha which are - orange/black (PIRHANA1.VTF), blue/white (PIRHANA2.VTF) or black/checkers (PIRHANA3.VTF).

There are in fact 79 different VTFs in i76 [117 in Nitro/Gold].

The names of VTFs are fairly self-explanatory; i.e. PIRANHA1.VTF is for the Piranha, CLYDSDL1.VTF for the Clydesdale, etc. (There is a list of all VTFs at the end of this guide).

Using Hex Workshop we can replace the reference to PIRANHA1.VTF in VPPIRNA1.VCF with a VTF file designed for another car, for example LEPRECN1.VTF, a paintjob designed for the Leprechaun...

To do this you need to use Hex Workshop to replace the reference to PIRANHA1.VTF with LEPRECN1.VTF

How can we tell which TMT is which?

Working out what each TMT stands for is reasonably easy:

Let's take 1PP1FTRT.TMT as an example and break it up into parts…

1PP1FTRT.TMT

The first number refers to which version of the paintjob it is for - this is "1" because its orange/black (i.e. PIRANHA1.VTF), if it was "2" it would be Blue/White (i.e. PIRANHA2.VTF) and "3" would be Black/Checkers (i.e. PIRANHA3.VTF)

1PP1FTRT.TMT

The next 2 letters refer to which Vehicle it is intended for. In this example it is PP.

1PP1FTRT.TMT

The next number refers to different views. There are three types...

"1" - Near view – This is the regular view you see of your own vehicle in F2 mode.

"3" - View from a distance – This is the view you see when zoomed out, or by another player.

"5" - View of the hood from the inside of the car – This speaks for itself.

I will explain these different views in more detail later.

1PP1FTRT.TMT

The final four letters tell us which section of the vehicle this part of the paintjob is for.

In this case it if "FTRT" which means Front Right. Each TMT uses a combination of 2 abbreviations to define its intended location.

The following abbreviations are used:

  • FT - Front
  • BK - Back
  • LT - Left
  • RT - Right
  • MD - Middle
  • TP - Top
  • UN - Under

Where are the TMTs?

The next move towards your custom paintjob is to find the TMT files that are listed in the VTF...

On first looking they do not appear to be within the ZFS. There are some files with the extension TMT, but not the ones listed in the VTF…

However they are there... to access them you need to use PACK76 to extract the following two files - PIRANA1T.PAK and PIRANA1T.PIX, and then extract the TMTs from these files.

Here's how you do that...

Exercise 3 (Extracting a TMT)

  • Extract the files PIRANA1T.PAK and PIRANA1T.PAK from I76.ZFS using Pack76
  • Double click on PIRANA1T.PIX (Make sure it is PIRANA1T.PIX not PIRANA1T.PAK)
  • Windows will ask you to choose the program you want to open it with
  • Scroll down the list of programs to NOTEPAD
  • Double click NOTEPAD

You will see the following text...

This is an index file that shows the contents of the file PIRANA1T.PAK

The number at the top is the number of files contained within PIRANA1T.PAK, in this case 24.

Then the names of the files are listed; each followed by 2 numbers

The first of these numbers is the file's offset in hex. The second number is the file's length

Using this information we can open PIRANA1T.PAK using Hex Workshop, and extract TMT files...

 

For this example we will extract 1PP1MDLT.TMT

  • Open PIRANA1T.PAK with Hex Workshop

It should look something like this…

  • From the menu choose Edit > Goto

A window appears asking for the offset

  • Type in the offset as listed next to 1PP1MDLT.TMT in PIRANA1T.PIX - 704
  • Click "GO"

This makes the cursor go to the value you put in, in this case 704

 

  • From the menu choose Edit > Select Block

A window appears asking for the size of the block…

  • Type in the size as listed in the PIX file - in this case 96
  • Click "OK"

The code that makes up 1PP1MDLT.TMT is highlighted

  • From the menu choose Edit > Copy

This copies the code onto the clipboard

  • From the menu choose File > New

A new file instance appears

  • From the menu choose Edit > Paste

This copies the code to the new file instance

  • From the menu choose File > Save As
  • When Prompted for File Name type 1PP1MDLT.TMT

We have extracted a TMT file.

[NOTE: You can extract TMT files from any file ending in "t.pak" with the method.

Also note Several different file types are "PAKed" like this within the ZFS, these include files with the following extensions - TMT, VQM, GEO and MAP.]

ORNGEVCL.CBK can be extracted from the ZFS with PACK76 and when opened in Hex Workshop looks like this...

There are no references to other files, but the file is quite large (65kb).

The extension CBK stands for Colour Bank.

There are various CBK files for various groups of textures, for example ORNGEVCL.CBK has information for Orange Vehicles, and REDBLD.CBK has information for Red Buildings, etc.

Presumably to save space, the VQM files are image files which refer to a large database of colour intersections stored in the CBK files.

Some of the values in the VQM files refer to a common palette that remains the same regardless of which CBK the VQM is referenced to.

This sounds complicated. Following the next exercise should help to explain...

Exercise 9 (Adapting a TMT and VTF file to accommodate a custom MAP)

The VQM and MAP files used for the outside of vehicles all have filenames that are 8 characters long (e.g. pp11mdl1). Because of this we need to rename our custom ZSY_.MAP file 12345678.MAP. [NOTE: The name of the MAP could be anything, as long as it is 8 Characters long, and different to the VQMs and MAPs that already exist.]

  • Go to the Addon Folder
  • Select ZSY_.MAP - it will become highlighted
  • Press F2
  • Type 12345678.MAP
  • Press Return

You have now renamed the file 12345678.MAP

To put this custom MAP onto the outside of the car you need to adapt a TMT file, and a VTF file.

  • Open 1PP1MDLT.TMT in Hex Workshop

(this is the file you extracted in Exercise 3. If you do not have it follow Exercise 3 again)

It should look like this...

 

The last 64 bytes refer to the following VQM files:

  • PP11MDL1.VQM
  • PP12MDL1.VQM
  • PP13MDL1.VQM
  • VW14MDL2.VQM

These are the references to the various images depicting the increasingly damaged states of the Piranha's left side.

 

We need to replace all 4 of these references with references to our custom file. For added realism you could make a series of different images, but for the purposes of this exercise we will replace ALL the references with 12345678. By doing this our image will remain constant regardless of hits.

  • Click Save

Summary

Any image can be imported into the game as long as:

    1. The I76.pal applied to it.
    2. It is saved in raw file format.
    3. It has the file extension MAP
    4. It has an 8-byte header inserted stating the image's dimensions in hex.

Using this information you can make your own custom paintjobs

 

Tips

Only make as many custom images as you need to. If no text if used on an altered side piece you can use one TMT for both sides (If text is used you need to make two because one will become mirrored).

Where possible use existing TMTs (even if they are designed for other vehicles). For example in the General Lee paintjob of a possible 23 TMTs I made only 5 custom ones. However as there is no red paintjob for the Dover Lightning in regular I76, most of the references to TMTs in the VTF were changed to existing ones for other cars (Here ones from the second version of the Cavera and the third version of the Sovereign). The General Lee paintjob is available for download from my page http://www.smidge.com/hoppo/

Some of the TMTs you can extract from I76.ZFS, such as BLADE101.TMT are used to have animated textures in the game (such as the cop car’s lights, helicopter’s blades, flames etc). By adapting these TMTs to refer to your custom MAP files you can make your own animated custom paintjobs.

Make sure the custom map you make has the same or similar dimensions to the VQM it is replacing. You can find the dimensions of any given VQM by opening it in Hex Workshop, and using Base Converter to translate the header. If the dimensions differ your map will be distorted when stretched on the vehicle.

The best way to ensure your custom map will properly fit is to use the original image as a guide. That is import it into Paint Shop Pro where you can recolor it, add text or a logo etc. However VQMs cannot be opened with Paint Shop Pro like MAP files can. To get round this you can replace a Roadmap VQM with the VQM you want to see, as we did in Exercise 6, and take a screen grab. The final exercise explains how to do this.
